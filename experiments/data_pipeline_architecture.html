SRE-Degrees — Data Pipeline Architecture

4 pipelines · 4 data sources · 19 real postmortems → 200K training trajectories → 42-incident eval benchmark

Dataset 1: Real Postmortems (seed)

  • • 19 hand-extracted from company blogs
  • • Fields: symptom, trap, fix, causal chain
  • • File: opensre-traj/real-incidents/catalog.md
  • • Feeds Pipeline 2 (RLVR) + Pipeline 3 (eval)

Dataset 2: SFT/DPO Training (200K)

  • • 150K SFT + 30K DPO + 20K rejections
  • • 15 incident types × 25 tools (templated)
  • • HuggingFace: quantranger/infra-ops-incidents
  • • NOT used for eval (no leakage)

Dataset 3: RLVR Corpus (319 scenarios)

  • • 83 synthetic + 114 real + 122 variants
  • • Interactive HUD v6 env (8 MCP tools)
  • • File: opensre-traj/out/trajectories.jsonl
  • • Reward: deterministic judge (P0 fix)

Dataset 4: Eval Benchmark (42 incidents)

  • • 8 simple + 14 cascade + 10 novel
  • • pass@1/2/5 with Wilson CIs + McNemar
  • • REx: 90% vs zero-shot: 23% (p<0.0001)
  • • File: scenarios/cidg/generated/*.yaml

Dataset 5: FIREBALL (transfer)

  • • 1,471 shards, CC-BY-4.0
  • • D&D → SRE trajectory converter
  • • HuggingFace: lara-martin/FIREBALL
  • • Claim 2 (BLOCKED on trained model)
