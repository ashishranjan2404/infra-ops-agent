# SRE-Degrees Benchmark — Data Sources & Provenance

## Overview

The SRE-Degrees benchmark is NOT sourced from an existing benchmark. It was built
from scratch for this research. There are 4 distinct datasets, each with a
different origin and purpose.

## Dataset 2: Scenario Specs (the benchmark)

**Count:** 62 YAML files (42 used for eval, 20 added during Ralph Loop)
**Location:** scenarios/cidg/ (10 original) + scenarios/cidg/generated/ (52 generated)
**Format:** YAML, each containing topology, root_cause, fault, observation, slo,
trap_actions, canonical_fix, assertions

### Origin of each subset:

### 10 original scenarios (hand-authored by Ashish)
Created early in the project. Each was manually written based on a real
postmortem. Files: 01-gcp-service-control.yaml through 30-singleton-node-notready.yaml

### 32 generated scenarios (auto-generated by Claude Code, first overnight)
Created by experiments/build_incidents.py. The generator read:
- The 19-postmortem catalog (Dataset 1)
- The sim engine schema (sim/spec.py)
- The danluu/post-mortems GitHub repo (700+ postmortem links)
- Existing YAML specs as format templates

Then generated valid YAML specs for:
- 8 simple incidents (40-47): redis-cache-flush, billing-disk-fill,
  auth-cert-expiry, checkout-bad-rollout, search-cpu-starve, ingest-fd-exhaust,
  media-oom-leak, payments-dep-revoked
- 14 cascade incidents (50-63): circleci-kubeproxy, datadog-systemd-cilium,
  incidentio-anetd, slack-tgw, slack-consul-cache, github-network-partition,
  github-proxysql, github-mysql-semaphore, cloudflare-zonemd, cloudflare-byzantine,
  cloudflare-bgp-reorder, github-dns-zone, launchdarkly-cold-cache,
  aws-kinesis-cell
- 10 novel incidents (70-79): facebook-bgp, cloudflare-leap-second,
  knight-capital, azure-leapyear, github-zk-splitbrain, firefox-addon-cert,
  cloudflare-waf-regex, gke-ip-exhaustion, kafka-poison-pill, conntrack-exhaustion

### 20 additional scenarios (Ralph Loop overnight, tasks A6/A11/A13/A14/J6)
Created during the second overnight run:
- A6: 10 new scenarios from danluu postmortems not in original catalog
- A11: 6 incident-pair scenarios (same root cause, different symptom)
- A13: 3 multi-fault scenarios (2 simultaneous faults)
- A14: time-pressure budget-limited variants
- J6: 1 novel NTP clock-skew split-brain scenario (true generalization test)

### Category split (for eval):
| Category | Count | Definition | Expected Winner |
|----------|-------|------------|-----------------|
| Simple | 8 | Symptom = root cause, no trap action | Harness-trained |
| Cascade | 14 | Symptom ≠ root cause, trap action exists | Fireball-trained |
| Novel | 10+ | Not in training set, tests generalization | TBD |
| **Eval total** | **42** | | |

### Validation (A16):
61 scenarios validated through the sim engine. 54 pass, 7 have broken canonical
fixes (4 authoring bugs, 3 missing SLO keys) — documented, not silently patched.

## Dataset 3: Synthetic Training Trajectories

**Count:** 200,000 total
**Location:** HuggingFace (quantranger/infra-ops-incidents)
**Generated by:** generate.py + generate_pathc.py
**NOT used for evaluation** — only for SFT/DPO training

| File | Count | Size | Purpose |
|------|-------|------|---------|
| incidents.jsonl | 150,000 | 434 MB | SFT trajectories (FIREBALL schema) |
| pairs.jsonl | 30,000 | 177 MB | DPO winner/loser preference pairs |
| rejections.jsonl | 20,000 | 60 MB | Labeled failures for RM/analysis |

**Source:** 15 incident types × 25-tool registry (tools_registry.json with trust
tiers). Templated generation, NOT derived from the 19 real postmortems.

**Important:** No train-on-test leakage. The 42-incident eval benchmark (Dataset 2)
and the 200K training set (Dataset 3) are completely separate.

## Dataset 4: FIREBALL D&D Trajectories

**Count:** 1,471 JSONL shards (100K-1M rows total)
**Location:** HuggingFace (lara-martin/FIREBALL), CC-BY-4.0
**Fetched:** Overnight Ralph Loop, task E2
**Purpose:** Cross-domain transfer experiments (Claim 2)

**Source paper:** Zhu et al., ACL 2023, arXiv:2305.01528

One real shard (1.7 MB, 111 turns) downloaded and converter validated — 49
trajectory records produced. The converter maps:
- observation <- before_utterances + combat_state_before
- actions <- commands_norm
- result <- automation_results
- next_observation <- combat_state_after

This is the "game data" that tests whether state-transition structure transfers
across domains (D&D → SRE).
