# B2 — Step 6: Implementation

## What I built (all real, task-namespaced; NO shared core files edited)

### `artifacts/mcnemar.py` (~280 lines, stdlib only)
Standalone paired-McNemar tool over **all** condition pairs.
- `to_binary(reward, threshold)` — pass@k-consistent binarization at the file's threshold.
- `mcnemar_table(a_bits, b_bits)` — 2x2 table (both_pass/both_fail/b01/b10), **exact**
  two-sided binomial p (`math.comb`, no scipy), continuity-corrected chi-square; raises on
  length mismatch.
- `holm_bonferroni(pairs, alpha)` — step-down multiple-comparison correction across the
  pairwise family; monotone-enforced adjusted p; authoritative `significant_holm` flag.
- `aligned_bits(...)` — incident-major, seed-ordered flatten so all conditions are paired
  episode-for-episode; raises KeyError (loud) if a condition is missing an incident.
- `family_incidents(...)` — adds an "overall" view + per-family (simple/cascade/novel).
- `analyze_file(...)` — for each family, C(k,2)=10 pair tables + Holm flags + pass rates.
- `main(...)` — CLI: `mcnemar.py R.json [...] [--out O.json] [--alpha] [--threshold]`,
  pretty table + JSON; exit 0 ok / 2 on broken input. Significance never gates exit code.

### `artifacts/test_mcnemar.py` (11 tests, stdlib unittest)
Binary thresholding, known discordant counts, n_disc=0 -> p=1.0, exact-binomial vs hand
calc, symmetry, length-mismatch raise, Holm correctness + monotonicity, alignment order,
missing-incident raise, end-to-end synthetic.

### `artifacts/mcnemar_pairwise_report.json` (REAL output)
Generated by running the tool on the two real cached pass@k result JSONs:
- A1: `experiments/ralph_outputs/A1/artifacts/full_pass_at_k_glm-5p2.json` (42 inc, 3 seeds)
- A2: `experiments/ralph_outputs/A2/artifacts/ablation_pass_at_k_deepseek-v4-pro.json` (30 inc, 5 seeds)

## Why a NEW tool rather than editing rex/run_ablation_v2.py
A2's `rex/run_ablation_v2.py` already contains a *narrower* embedded McNemar (REx vs each
of 4 controls, overall + per family only — 4 directed comparisons). The brief forbids
editing shared core files, and B2's mandate is the **full pairwise matrix** (all 10 unordered
pairs) the paper claims but never ran, plus Holm correction A2 omitted. So B2 ships a
standalone, reusable, dependency-free tool that ingests the same artifacts read-only.

## Cross-validation against A2
For deepseek overall, B2's rex-vs-control discordant counts and rex-win counts reproduce
A2's `ablation_v2_mcnemar_deepseek-v4-pro.json` **exactly** (zero_shot 100/99, best_of_n
88/88, retry_realistic 89/88, rex_no_oracle 91/91). This confirms the core math is correct
and that B2 generalizes A2 to the complete matrix.
