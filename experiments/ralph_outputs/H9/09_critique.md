# H9 — 09 Critique (honest)

## What a reviewer will attack
1. **It's a presentation artifact, not new science.** True. H9 produces no new runs; it
   surfaces A1/A2/E3 numbers in a webpage. Its value is communication and auditability, not
   a new result. I do not claim otherwise.
2. **Cross-family ranking is still a category error.** Row 1 (glm-5p2, 42 incidents) sits above
   row 8 (qwen3-8b, 14 cascades) in the default pass@1 sort. The family tag + filter mitigate
   the misread, but a careless reader can still screenshot "rank 1 = 0.90" out of context. A
   stricter design would refuse a global rank and only show within-family/within-model deltas.
3. **The headline lift (0.23 → 0.90) is the same number A1 already flagged as suspiciously high.**
   The board inherits A1's open question of whether the benchmark is too easy for REx; it does
   not re-litigate it. A reviewer wanting harder evidence gets none here.
4. **Thin coverage.** Only 3 source tasks, 2-3 models, 3 families. A real public leaderboard
   would have many models and a stable held-out incident set. This is a credible *prototype*,
   not a populated community board.
5. **`file://` foot-gun.** The page only works when served over HTTP. We handle it gracefully
   (error message) and the verifier uses HTTP, but a non-technical visitor double-clicking the
   file gets the error path, not the board.

## What's weak / missing
- No automatic regeneration: the JSON is hand-assembled from the source JSONs, not generated by
  a script that reads A1/A2/E3 directly. The verifier *catches* drift but doesn't *prevent* it.
  A `build_leaderboard.py` that derives the JSON from the source artifacts would be stronger.
- pass@5 column is near-useless at these seed counts (kept for schema completeness, flagged).
- No statistical annotation of significance between adjacent rows on the page itself (the McNemar
  / disjoint-CI evidence lives in A1/A2, only summarized in notes).

## What's genuinely solid
- The data/render split + HTTP load proof means "renders a pass@k table from JSON" is verified,
  not asserted.
- Every cell is provenance-tagged and machine-checked against its source task — drift fails CI.
- The blocked Fireball arm is shown honestly; no fabricated numbers anywhere.
- Zero dependencies, no shared-core edits, fully parallel-safe.
