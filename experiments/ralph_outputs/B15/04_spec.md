## Script: `artifacts/gen_comparison.py`
Pure stdlib. Functions:

- `load_our(a1_path, a2_path) -> dict`
  Reads A1/A2 JSON. Pulls `by_condition[cond][family]["pass@1"|"p1"]`, `ci95|ci`, `n`.
  Robust to either key name. Returns the `our_pass_at_1.json` structure; also writes it.
- `load_sregym(path) -> dict` ŌĆö loads the checked-in reported JSON.
- `fmt_pct(x) -> str` ŌĆö `0.607 -> "60.7%"`, `None -> "ŌĆö"`.
- `render_table(our, sregym) -> str` ŌĆö returns Markdown. Two tables:
    1. **Cross-benchmark headline**: our 5 conditions (overall pass@1, both models) vs SREGym
       E2E band (minŌĆōmax across agents, no-noise), with an `attempt regime` column.
    2. **Loose familyŌåöpartition analogy**: our per-family pass@1 (rex + a single-run-ish cond)
       beside SREGym per-partition E2E (one representative agent), clearly labeled "loose analogy".
- `main()` ŌĆö argparse `--a1 --a2 --sregym --out-table --out-our`; writes both files; prints table.

### CLI contract
```
python3 gen_comparison.py \
  --a1 ../../A1/artifacts/full_pass_at_k_glm-5p2.json \
  --a2 ../../A2/artifacts/ablation_pass_at_k_deepseek-v4-pro.json \
  --sregym sregym_reported.json \
  --out-table comparison_table.md \
  --out-our our_pass_at_1.json
```

## Test cases (in `gen_comparison.py` under `--selftest`)
1. `fmt_pct(0.6068) == "60.7%"`, `fmt_pct(None) == "ŌĆö"`, `fmt_pct(1.0) == "100.0%"`.
2. `load_our` returns 5 conditions for A1 and finds `rex` overall p1 Ōēł 0.897 (┬▒0.001).
3. `load_sregym` has 8 leaderboard rows and `n_problems == 90`.
4. `render_table` output contains "SREGym", "REx", "pass@1", and a non-equivalence marker.
Exit non-zero if any assert fails.
